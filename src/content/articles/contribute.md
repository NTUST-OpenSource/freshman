:::tip[不會寫程式也能參與嗎？]
完全沒問題！修錯字、補資訊等簡單的內容修改，全程都能在網頁上點擊操作，也不用背程式碼指令。我們備有詳細的貢獻指南，跟著引導步驟走就能輕鬆完成貢獻。

## 為什麼要開源？

新生懶人包最大的敵人，往往不是「寫不出來」，而是「沒人維護」。校內制度年年改、連結年年壞，如果只靠少數幾個人盯著，只要兩三年沒有更新，就會淪為過期資訊。

為了讓專案的存續**不依賴特定個人**，確保學長姐畢業後，學弟妹依然能接手傳承，本站從一開始便以 GitHub 公開專案的形式運作：

:::steps
1. **內容即檔案**：每篇文章都是一個獨立檔案，改內容就像改文件，不需要申請後台帳號。
2. **人人能提案**：任何在校生都能提出修改，補一句話、改一個錯字都大歡迎。
3. **品質有把關**：提案不會直接上線，經維護者確認後才會併入，確保資訊正確性。
4. **歷史全保留**：誰在什麼時候改了什麼皆有紀錄，不小心改壞了隨時能還原。
:::

這也是我們選擇 GitHub 的原因，它完美解決了多人協作會遇到的痛點：

| 協作痛點 | GitHub 的解決方案 |
|---|---|
| 誰改了什麼 | 每次修改都是一筆紀錄，附帶說明與作者，隨時查得到。 |
| 改壞了怎麼辦 | 所有版本妥善保存，可以隨時回溯到任何一個時間點。 |
| 品質怎麼把關 | PR（Pull Request）機制：先審核再合併，不是先上線再說。 |
| 問題怎麼追蹤 | Issue 功能讓每個錯誤都有專屬的公開討論空間，不怕沉沒在私訊裡。 |

## 你可以貢獻什麼？

只要是對學弟妹有幫助的資訊，我們都歡迎：

- **回報錯誤**：內容寫錯、連結失效、資訊已過期。
- **補充內容**：某個主題講得不夠清楚，或缺少了某些環節。
- **各系專屬資訊**：系學會運作、選課慣例、必修避雷，特別歡迎在校生現身說法。
- **經驗分享**：你曾經踩過的坑，就是下一屆最需要的避險指南。
- **技術改善**：版面優化、無障礙設計、網站效能調整。

:::info[不確定該不該提？]
提出來就對了！與其自己猶豫不決，不如直接開個 Issue 讓大家一起討論。我們很樂意收到各種建議，也不希望錯過任何能讓網站變得更好的點子。
:::

## 參與的兩種方式：Issue 與 PR

參與貢獻主要分為兩種管道，你可以視情況選擇最適合的方式：

### 1. 發現問題，請大家討論 -> 開 Issue
Issue 就像是專案的「留言板」。當你發現網站有問題，但不知道正確答案，或者不方便自己動手改時，只要把問題說清楚即可。

:::steps
1. 前往 [Issues 頁面](https://github.com/NTUST-OpenSource/freshman/issues)，點擊右上方 **New issue**。
2. 根據情況選擇模板：**內容錯誤**、**內容補充** 或 **網站問題**。
3. 標題請寫出**具體問題**（例如：「住宿篇的報修表單連結已失效」），避免只寫「有問題」。
4. 依照模板填寫。請明確指出是**哪一篇**、**哪一段**出錯；若知道正確資訊，也請一併附上。
5. 送出！後續的處理進度都會在該則 Issue 底下進行討論。
:::

### 2. 我已經改好了，請審核 -> 發 PR (Pull Request)
如果你明確知道怎麼修正（例如改錯字、更新失效連結），直接發送 PR 是最快的途徑。這代表你已經把檔案改好了，請維護者過目並更新到網站上。

*   **建議直接發 PR 的情況：** 錯字修正、更新失效連結、明顯過期的資訊（如日期、金額）、補充你的第一手實際經驗。
*   **建議先開 Issue 討論的情況：** 新增一整篇文章、大幅重寫既有內容、調整網站底層功能、不確定資訊是否正確時。

發起 PR 會稍微牽涉到 Git 與 GitHub 的基本操作，會用到一點點技術能力。如果你是程式新手，只要有興趣挑戰看看，歡迎參考我們準備的詳細說明頁面，跟著教學一步步嘗試！

::card[查看詳細貢獻指南]{href="https://github.com/NTUST-OpenSource/freshman/blob/main/CONTRIBUTING.md" desc="內含兩種操作方式的逐步教學、分支命名規範與送出前的檢查清單。"}

:::tip[送出 PR 之後呢？]
維護者通常會幾週內進行初步審核。如果內容需要微調，維護者會留言討論，這**不是否定**你的貢獻，而是為了讓內容達到上線標準。確認無誤後，維護者就會將你的修改合併上線！
:::

## 寫作基本守則

在開始動手前，請留意以下幾項基本原則：

- **嚴禁抄襲與直接複製**：本站為獨立創作，任何參考資訊都必須**經過查證並用自己的話重寫**，直接複製他站內容的 PR 將不予合併。
- **排版規範**：全站使用繁體中文。標點符號請用全形，數字與英文字母用半形，且中英文/數字之間需**加上一個半形空白**。
- **禁用 Emoji**：為維持風格統一，內容、介面與 commit 訊息皆禁止使用 emoji，請一律使用站內既有的 SVG 圖示。
- **標註時效性**：制度相關內容，請註明是「哪一學年」的規定（例如：113 學年度），方便未來核對是否適用。
- **保持客觀保留**：若遇到不確定的資訊，請寫「依當年最新公告為準」並附上官方連結，避免提供錯誤的絕對資訊。

## 需要協助嗎？

* **公開討論**：關於內容問題或網站錯誤，最有效率的方式是直接開 [GitHub Issue](https://github.com/NTUST-OpenSource/freshman/issues) 討論。
* **加入維護團隊**：如果你對長期維護專案有興趣，歡迎開 Issue 說明你想參與的部分，維護者會主動與你聯繫。
* **私下聯絡**：如果有不便公開的事項，每篇文章末尾的貢獻人卡片皆附有該篇維護者的聯絡信箱，歡迎來信。

:::tip
第一次開源貢獻難免會遇到卡關，這非常正常！遇到任何問題隨時開 Issue 發問，這裡的大家都很樂意協助你。
:::
