## 二、各年份行政語法差異（重點發現）

1. **殘留編號前綴**：110–114 的檔案中，多事項事件已被前人拆開，但保留了「N.」前綴（如 111 的「3.暑假結束」、114 的「2.暑假結束」）。115 則是未拆的原始多事項格式（「1.xxx 2.yyy」），由腳本自動拆分。**單獨出現的「N.」前綴是雜訊，應一律去除。**
2. **範圍措辭在 113 學年改版**：110–112 幾乎不用「(至X月X日截止)」；113 起大量採用（113:12 筆、114:16、115:16）。舊年份的範圍多半寫成獨立的「XX開始」「XX結束」兩筆事件。
3. **民國年出現在範圍內**：「寒假開始(至113年2月16日止)」使用民國年，範圍解析需支援。
4. **標點混亂且跨年不一致**：全形/半形括號混用，甚至同一標題內半開全閉（「國慶日(放假一天）」）；逗號有「，」與「,」兩種；括號前空白時有時無（「中秋節(放假一天)」vs「中秋節 (放假一天)」）。
5. **period 例外全六年僅 1 筆**（115 的「臺灣光復…補假。期中考試開始…」），人工處理成本極低。
6. **digits 例外佔 29–48%**，主要構成：日期範圍句（至…截止/止）、補上班（補X月X日之調整放假）、「第N次校務會議」、學年度數字。多數可由範圍解析規則消化後解除例外。

## 三、跨年份重複事件

## 五、一致性斷言草案（待確認或修改）

- A1. 標題開頭殘留編號「N.」一律去除（不論是否觸發拆分）。
- A2. 標點正規化：括號統一半形、逗號統一「，」、去除括號前空白、去除結尾「。」。
- B1. 「(至X月X日截止)」「(至X年X月X日止)」解析為 `end` 迄日，支援民國年與跨年推定；解析成功後自標題移除該括號段，digits 例外解除。
- B2. 「(放假一天)」等時長說明：自標題移除，資訊已由 start/end 表達。
- B3. 「(若教師另有訂定其他日期者，從其規定)」等附註：移入 `note`，自標題移除。
- C1. 「第N次校務會議」：數字保留，tag 行政，建議 `hidden: true`（與新生無關）。
- C2. 「補X月X日之調整放假」：數字保留於標題（語意必要），tag 假期，解除例外。
- D1. 經 B/C 規則處理後仍含數字者，維持 digits 例外交人工。
- E1. period 例外一律人工拆分。
