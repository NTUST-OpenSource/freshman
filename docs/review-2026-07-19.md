## CRITICAL

### C1. `data-dept` 在每次頁面導航後遺失，系別功能全站失效
`src/layouts/Base.astro:51-53`
ClientRouter 的 `swapRootAttributes()` 會清掉 `<html>` 上所有非 astro 屬性再套用新頁的靜態屬性（已比對 `astro/dist/transitions/swap-functions.js:23-31`），而設定 `data-dept` 的 `is:inline` script 被 router 以 textContent 去重、只在首次載入執行一次，之後永不重跑，也沒有任何 `astro:after-swap` 補救。
情境：文章頁選「資工系」→ 點任一站內連結 → `html:not([data-dept='csie'])` 成立，**所有 csie 專屬區塊對資工系使用者隱藏**；再導航一次後 `sync()` 讀到空屬性，選單 label 退回「全部」，與 localStorage 矛盾。
建議：模組 script 頂層一次性掛 `astro:after-swap` 還原 `dataset.dept`，`sync()` 改以 localStorage 為準。

### C2. 系別下拉選單鍵盤完全無法操作（WCAG 2.1.1）
`src/layouts/Base.astro:79-85、138-146`
選項是 `li role="option" tabindex="-1"`，只綁 click：無方向鍵、無 Enter／Space、無焦點管理。鍵盤使用者開啟選單後 Tab 直接跳過所有選項，永遠選不了系別，`:::dept` 條件內容對鍵盤使用者形同不存在。
建議：補 keydown（上下鍵＋`aria-activedescendant`＋Enter），或直接改原生 `<select>`。

## HIGH

### H1. `--cat` 全站從未定義，北捷分類色系統整個是死碼
`src/pages/article/[slug].astro:30`、`src/styles/tokens.css:15-18`
`cat-${category}` class 有輸出，但全 repo 沒有任何 `.cat-course { --cat: var(--c-course) }` 之類的規則（`--cat:` 定義零筆、`var(--cat` 使用 18 筆），chip、h2 色條、目錄路線、steps、tabs 選中色全部 fallback 成臺科紅——「分類色＝北捷路線色」的核心設計語言實際沒生效。
注意：補上四條規則後有連帶對比度地雷——白字 chip 在 `--c-course` 4.33:1、`--c-info` 3.58:1（小字需 4.5:1），tabs 選中 `--c-info` 3.32:1，需一併調深這兩色。

### H2. 複製按鈕會把「複製」二字一起複製進剪貼簿
`src/layouts/Base.astro:164、172`
按鈕被 `appendChild` 進 `<pre>` 內，點擊時讀 `pre.textContent` 串接所有後代文字——每次複製結果尾端多「複製」（1.4 秒內連點則是「已複製」）。
建議：改抓 `pre.querySelector('code')?.textContent ?? ''`。

### H3. `==mark==` 正規式在內容含 `=` 時跨界污染鄰近文字
`src/plugins/remark-custom.mjs:244`
實測 `==學分上限=25== 之後的正常文字 ==真重點==` → 正常敘述被螢光、兩個原意標記全毀、殘留字面 `==`。
建議：匹配後檢查邊界，失敗時整段還原字面文字。

### H4. 落單 `:::tab` 的 fallback 內容整段隱形
`src/plugins/remark-custom.mjs:114-116`＋`src/styles/markdown.css:410`
fallback 給了 `tabs__panel` class，但 `.tabs__panel { display: none }` 是全域選擇器。外層 tabs 冒號數打錯時（`:::tabs` 配 `:::tab`），第二個 tab 變落單 panel 完全看不見，頁尾還漏出字面 `<p>:::</p>`——與註解「當一般區塊呈現」意圖相反。
建議：落單 tab 用獨立 class（或 CSS 改 `.tabs .tabs__panel`），並對等冒號誤寫發 build 警告。

### H5. tabs 不擋分頁數上限，第 7 個分頁起點擊後內容區全空白
`src/plugins/remark-custom.mjs:203-231`＋`src/styles/markdown.css:430-445`
CSS `:has()` 規則只列舉到 6，serializer 不驗數量。
建議：`items.length > 6` 時 build 失敗，SPEC 載明上限。

### H6. `::::tabs` 內非 `:::tab` 子節點被靜默丟棄
`src/plugins/remark-custom.mjs:205、226`
tabs 內夾說明文字 → 該段從輸出完全消失，作者無任何提示。
建議：保留輸出於 panels 前，或拋 build 錯誤。

### H7. div 容器內的表格不會被包橫向捲動層
`src/plugins/rehype-post.mjs:10`
防重複包裝的判斷寫成 `parent.tagName !== 'div'`，誤傷所有 div 型容器：`:::dept`、tabs 分頁、steps 內的表格全部不包 `.table-scroll`，寬表在窄視窗直接爆版，`:::table{vlines}` 也掛不上。
建議：改判斷 `parent.properties?.className?.includes('table-scroll')`。

### H8. 行事曆更新時間在 UTC build 機上少一天
`src/components/Calendar.astro:62-70`
`parsedAt`（`2026-07-19T07:58:09+08:00`）用 `toLocaleDateString('zh-TW')` 未指定時區；已實測 `TZ=UTC` 輸出「2026年7月18日」。Cloudflare Pages CI 是 UTC，凌晨至上午的 parsedAt 必錯。
建議：options 加 `timeZone: 'Asia/Taipei'`。

### H9. `calendarEnd` 有兩種慣例，months 計算假設單一慣例，未來會整月消失
`src/components/Calendar.astro:38-44`（成因 `scripts/parse_ics.py`）
months 迴圈把 calendarEnd 當「排他月初」，但管線直接照抄 ics 的 X-CALEND：實查六檔中 111＝`2023-07-31`、112＝`2024-08-30`、113＝`2025-07-31` 是「月末日」慣例。目前僅因排序後最大值恰為 115 的 `2027-08-01` 才沒爆；若 116 沿用月末慣例（`2028-07-31`），**整個 2028 年 7 月（期末考月）消失**。
建議：months 計算對「end 日非 1 號」把該月含入，或在管線正規化並於 calendar-sync 檢核斷言。
